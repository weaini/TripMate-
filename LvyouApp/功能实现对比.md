# 爱游吧旅游服务平台 — 功能实现对比

基于功能层次图中的 6 大模块，对当前系统（后端 LvyouApp + 前端 LvyouAppUNi）的实现情况做逐项对比。

---

## 1. 用户信息模块

| 功能 | 实现情况 | 说明 |
|------|----------|------|
| **登录/注册** | ✅ 已实现 | 后端 `AuthController`：`/api/auth/login`、`/api/auth/register`；前端 `Login.vue`、`Register.vue`，路由守卫与 token 处理完整。 |
| **用户资料管理** | ✅ 已实现 | 后端 `UserController`：`/api/users/me`、`/api/users/profile`（PUT）、`/api/users/avatar`；前端 `Profile.vue`、`EditProfile.vue`、`ProfileSettings.vue`、`ChangePassword.vue`。 |
| **用户权限管理** | ✅ 已实现 | 后端 `SecurityConfig` + `@PreAuthorize`，角色 TOURIST/EXPERT/ADMIN；前端路由 `requiresAuth`、`requiresAdmin`、`requiresExpert`，权限控制一致。 |
| **关注系统** | ✅ 已实现 | 后端 `UserController`：关注/取关、关注列表、粉丝列表（`/api/users/{id}/follow`、`following`、`followers`）；前端在个人中心等可展示关注/粉丝（依赖接口调用）。 |

---

## 2. 动态管理模块

| 功能 | 实现情况 | 说明 |
|------|----------|------|
| **动态发布管理** | ✅ 已实现 | 后端 `PostController`：创建、删除、上传/删除图片、点赞/取消；前端 `Posts.vue`、`CreatePost.vue`、`PostDetail.vue`、`MyPosts.vue`。 |
| **内容审核管理** | ✅ 已实现 | 后端 `AdminController`：`getPendingPosts`、`approvePost`、`rejectPost`；动态实体 `PostStatus.PENDING/APPROVED/REJECTED`；前端 `AdminPostReview.vue`（动态审核页）。 |
| **评论功能** | ✅ 已实现 | 后端 `CommentController`：动态评论创建、分页列表、子评论、点赞/取消、删除；前端在 `PostDetail` 等使用评论组件。 |
| **分类管理** | ✅ 已实现 | 后端 `Post` 实体有 `PostType`（DYNAMIC/STRATEGY/ACTIVITY），`PostController.getPosts` 支持 `type` 等筛选；前端列表可按类型筛选。 |

---

## 3. 活动管理模块

| 功能 | 实现情况 | 说明 |
|------|----------|------|
| **活动创建** | ✅ 已实现 | 后端 `ActivityController`：创建、更新、上传图片；前端 `CreateActivity.vue`、`EditActivity.vue`，需达人/管理员权限。 |
| **活动报名** | ✅ 已实现 | 后端 `joinActivity`、`cancelJoinActivity`、`checkParticipation`；前端 `ActivityDetail.vue` 等有报名/取消入口。 |
| **参与者管理** | ✅ 已实现 | 后端 `approveParticipant`、`rejectParticipant`，获取参与者列表；前端 `ActivityParticipants.vue`（参与者管理页）。 |
| **活动聊天室** | ✅ 已实现 | 后端 `WebSocketConfig` + `WebSocketController`，活动评论通过 WebSocket 推送（`/topic/activity/{id}/comments`）；前端 `ActivityChatSimple.vue`，路由 `/activities/:id/chat`。 |

---

## 4. 攻略管理模块

| 功能 | 实现情况 | 说明 |
|------|----------|------|
| **攻略发布** | ✅ 已实现 | 后端 `GuideController`：创建、更新、删除、封面上传；前端 `CreateGuide.vue`、`EditGuide.vue`、`GuideDetail.vue`、`MyGuides.vue`。 |
| **攻略分类** | ✅ 已实现 | 后端 `getGuides(type, status, sortBy)` 支持按类型、状态、排序；前端 `Guides.vue` 等可做分类/筛选。 |
| **攻略评论** | ✅ 已实现 | 后端 `GuideController`：创建评论、分页获取、点赞/取消、删除；前端攻略详情页集成评论。 |
| **攻略推介** | ⚠️ 部分实现 | 后端有 `type`、`sortBy`（如按热度、时间），可间接实现“热门/最新”推介；无单独的“精选/推荐”接口或 FEATURED 标记（Strategy 实体有 FEATURED，guides 表未体现）。若需“编辑精选”需后端增加字段或接口。 |

---

## 5. 积分系统模块

| 功能 | 实现情况 | 说明 |
|------|----------|------|
| **积分发放功能** | ✅ 已实现 | 后端 `PointsService`：发帖/发布攻略/创建活动 +10 积分等；在 `PostServiceImpl`、`GuideServiceImpl`、`ActivityServiceImpl` 中调用。 |
| **积分消费** | ✅ 已实现 | 后端评论、点赞等扣积分（如评论扣 1 积分），`PointsService.deductPoints`、`spendPointsForComment` 等；前端评论前会校验积分（`PointsDisplay`、`CommentList`）。 |
| **积分记录** | ✅ 已实现 | 后端 `UserController` 提供 `GET /api/users/me/points-records` 分页查询；前端 `PointsRecords.vue` 积分明细页展示流水，个人中心积分区域点击可进入。 |

---

## 6. 系统管理模块

| 功能 | 实现情况 | 说明 |
|------|----------|------|
| **用户管理** | ✅ 已实现 | 后端 `AdminController`：`getUsers(role, keyword)`、`updateUserStatus`、`updateUserRole`；前端 `AdminUserManagement.vue`，支持按角色/关键词查询与状态、角色修改。 |
| **达人审核** | ✅ 已实现 | 后端 `AdminController`：`getExpertApplications`、`approveExpertApplication`、`rejectExpertApplication`；`ExpertController` 提供申请与我的申请状态；前端 `AdminExpertReview.vue`、`ExpertApplication.vue`。 |
